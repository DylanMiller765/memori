## Critical Anti-Patterns (BLOCKING — must understand before resuming)

| # | Anti-Pattern | Manifestation | Prevention |
|---|---|---|---|
| 1 | `.preferredColorScheme(.dark)` alone doesn't propagate `Color.primary` resolution through TabView | Goals page rendered near-black headline/titles on dark bg because `AppColors.textPrimary` (= `Color.primary`) resolved to light-mode variant despite parent's `.preferredColorScheme(.dark)` | Combine `.preferredColorScheme(.dark)` AND `.environment(\.colorScheme, .dark)` at OnboardingView root. Both modifiers are now in place at `OnboardingView.swift:131-132` |
| 2 | Named asset colors (`AppColors.pageBg`) adapt to system theme even when child views override colorScheme | Welcome page rendered cream background in light mode despite welcomePage having `.preferredColorScheme(.dark)` — outer ZStack's `AppColors.pageBg` resolved to its light-mode variant | Use hardcoded `OB.bg` (the OB token, not the named asset) at OnboardingView root for non-Quick-Assessment pages. Done at `OnboardingView.swift:73` |
| 3 | GSD executor agents may rewrite components more aggressively than expected | Plan 01 executor replaced `EvidenceMiniSlip` with `PainReceiptSlip` and changed several private struct names. Caused identification confusion when re-fixing | Future executor prompts must explicitly say: "Edit existing struct in place — do NOT rename, do NOT replace with a new struct, do NOT introduce new private types unless the plan specifies." |
| 4 | iOS device deployment is single-threaded — parallel wave execution doesn't apply | Plans 01-03 are in Wave 1 with different file targets, but they all build/install to the same physical iPhone. Spawning 3 parallel executors would race for `xcrun devicectl install` | Run plans sequentially when iOS device is involved, even if wave structure allows parallelism. Use parallelism only when targeting different simulators or different build artifacts |
| 5 | Hard reset to discard GSD work also discarded uncommitted in-flight fixes | When user said "reset to before GSD," I lost the in-progress dark-mode fix. Had to stash, reset, pop. | Before any `git reset --hard`, always save uncommitted work via `git stash push` first. Then pop after. |
