# v1.1 Competitive Edge Features — Design Doc

## Context
Based on competitive research into Lumosity and Elevate 1-2 star App Store reviews, we identified 3 high-impact features that directly address the top user complaints about brain training apps: hidden cancellation, opaque scoring, and no feedback on mistakes.

## Feature 1: "How to Cancel" Link in Settings

### What
Add a "How to Cancel" row in the Settings > About card that opens Apple's subscription management page.

### Why
Both Lumosity and Elevate get destroyed in reviews for making cancellation hard to find. Adding this builds trust and signals we're not trying to trap users.

### Design
- **Location:** `SettingsView.swift` aboutCard, between "Manage Subscription" and "Restore Purchases"
- **Only visible to Pro users** (free users have nothing to cancel)
- **Style:** Same as existing about rows — `xmark.circle.fill` icon in gray, "How to Cancel" text, chevron right
- **Action:** Opens `https://apps.apple.com/account/subscriptions`
- **No confirmation dialog** — just opens the URL directly

- `MindRestore/Views/Settings/SettingsView.swift` — add row in aboutCard

## Feature 2: Brain Score Explainer Bottom Sheet

### What
An ⓘ info button on the BrainScoreCard that opens a bottom sheet explaining how brain score is calculated.

### Why
Lumosity users complain about "rigged scoring." Transparency kills this perception before it starts.

## Feature 3: Correct Answer Feedback on Wrong Answers

### What
Show the correct answer briefly when users get a question wrong, so they learn from mistakes.

### Why
Elevate's #1 learning complaint: "When you get something wrong, no explanation of why." This is the easiest way to make our games feel educational, not just punishing.

### Games & Approach

#### ColorMatch
- **Current:** Red flash for 0.45s, moves on
- **New:** Extend wrong feedback to 0.8s. Add text below the word: "Correct: [color]" displayed in that color's actual color
- **Implementation:** Add conditional text in `playingView` when `showFeedback && feedbackColor == red`. Increase `asyncAfter` delay from 0.45 to 0.8 on wrong answers only.
- **File:** `MindRestore/Views/Exercises/ColorMatchView.swift` + `ColorMatchViewModel`

#### SpeedMatch
- **Current:** X overlay for `feedbackDelay`, moves on
- **New:** On wrong answer, show a small comparison: previous symbol + current symbol with "Match" or "No Match" label. Extend feedback to 0.6s on wrong only.
- **Implementation:** Add `lastWrongPrevSymbol` and `lastWrongIsMatch` to ViewModel. In feedback state, show comparison overlay below the main symbol circle.
- **File:** `MindRestore/Views/Exercises/SpeedMatchView.swift` + `SpeedMatchViewModel`

#### DualNBack
- **Current:** Just haptic wrong, trial auto-advances on timer
- **New:** After wrong position/letter tap, briefly flash what was N steps back. Small overlay text at bottom: "Position N-back: cell [X]" or "Letter N-back: [L]". Show for ~1s during the inter-trial interval.
- **Implementation:** Add `lastFeedback` state to ViewModel/View. In `tapPosition()`/`tapSound()`, when wrong, store what the N-back value was. Show overlay that auto-hides.
- **File:** `MindRestore/Views/Exercises/DualNBackView.swift` + `DualNBackViewModel.swift` + may need to expose history from `DualNBackEngine`

## Out of Scope
- ProspectiveMemory, ActiveRecall, SpacedRepetition, MemoryPalace, MixedTraining — removed from app
- Chunking — already has per-character feedback
- Visual Memory — already shows correct pattern on wrong
- Sequential Memory — already shows correct sequence
- Math Speed — already has "Review Mistakes" section
- Reaction Time — nothing to correct
