| Rank | Brain Score Range | Color |
|------|------------------|-------|

| Silver | 300-499 | Steel grey (#A6ADB8) |

### Display
- **Text-only badges** with colored glow — no custom art assets
- `RankPill`: uppercase rank name in rank color, subtle glow shadow, capsule background with 12% opacity fill + border
- Visible on: Profile (next to name + progress bar), Leaderboard (next to each player on Brain Score view)

### How Rank Works
- Rank is derived from current Brain Score — `PlayerRank.from(brainScore:)`
- Brain Score already exists and is calculated from game performance across all domains
- Brain Score decays naturally if user stops playing (existing decay system)
- Rank goes up when Brain Score improves, goes down when it decays

## 2. Seasonal Rank Resets

### Mechanic
- Ranks reset on the **1st of every month** (or bi-weekly — TBD based on user behavior)
- On reset: user's Brain Score drops by **200 points** (clamped to 0 minimum)
- This knocks most users down 1-2 ranks
- Users must play games to climb back up
- Reset happens on app open after the reset date

### Implementation
- Store `lastRankResetDate` in UserDefaults
- On app launch, check if current date is past the next reset date
- If so: subtract 200 from latest BrainScoreResult, create a new BrainScoreResult with the reduced score, update `lastRankResetDate`
- Show a "Season Reset" splash screen: "New season! Your rank has been reset. Climb back up."
- Analytics event: `rank_season_reset(previousRank:, newRank:, previousScore:, newScore:)`

### Edge Cases
- User hasn't opened app in 3 months: only apply ONE reset (not 3), because decay already handles long absences
- User has no Brain Score yet: skip reset
- Score can't go below 0

## 4. Random Dopamine Mechanic (TBD)

### Requirements
- Must be **unpredictable** — user can't anticipate when it happens
- Must create **craving** — user wants it to happen again
- Must NOT require custom art assets
- Must fit the competitive brain training context

### Options Under Consideration

**Option A: Mystery Stats**
After each game, ~20% chance to show a rare/impressive stat card:
- "Top 3% reaction time globally today"
- "You beat 96% of Diamond players on this game"
- "Fastest Number Memory solve in the last hour"
- "Your Visual Memory is better than 89% of users your age"

Why it works: social comparison is inherently addictive, the stat is real data (not fake), and you never know if you'll get one or which one it'll be. Some stats are more impressive than others, creating a natural rarity system.

Implementation: after game complete, roll 20% chance → if hit, pick a random stat template → fill with real data from Game Center leaderboard position or personal bests → show as a toast/card that appears on the results screen.

**Option B: Score Streak Bonuses**
Hidden multiplier that builds when you score above your average:
- 2 games above avg → "On fire" badge appears
- 3 games → "Unstoppable"
- 5 games → "Brain Surge — Peak Performance"
- Streak breaks when you score below average
- No XP bonus, just the badge/recognition

Why it works: you don't know when you'll get on a streak because it depends on your actual performance. The badge appearing unexpectedly feels rewarding. Other players seeing "Unstoppable" next to your name is social flex.

**Option C: Daily Mystery Challenge**
One random challenge per day, revealed after your first game:
- "Score 300+ on Reaction Time"
- "Win 3 games in a row"
- "Beat your personal best on any game"
- Completing it gives a small Brain Score bonus (+10-20)
- Challenge is different every day and you don't know what it is until you start playing

Why it works: daily surprise element, achievable goals that vary, small but meaningful reward.

### Decision
TBD — need to pick one and test. Option A is lowest effort and most data-driven. Option C is most gamified. Can prototype all three and A/B test.
